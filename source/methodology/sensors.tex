\startsection[title={Sensores}]
	A princípio, as variáveis medidas podem apresentar elevada colinearidade. Entretanto, o método de análise para a produção de \getbuffer[sens] baseia-se em um modelo de caixa-preta (LSTM), de modo que o sistema de medição é projetado para monitorar o maior número possível de variáveis acessíveis pelos sensores disponíveis. As imagens de todos os sensores utilizados são apresentadas no Apêndice A.

	\startitemize
		\starthead{Temperatura}
			\indentitem A temperatura é a variável medida com maior precisão dentre as disponíveis. O sensor utilizado comunica-se com o controlador por meio do protocolo {\emph one-wire}, que requer apenas uma linha de dados.

			O DS18B20 possui faixa de medição de \math{\left\lbracket -55, 120 \right\rbracket \celsius} e precisão de \math{\pm 0.5 \celsius} nessa faixa de operação.
		\stophead

		\starthead{Potencial de Oxirredução}
			\indentitem O potencial de oxidação-redução (ORP) representa o estado de oxidação da solução e fornece informações sobre seu equilíbrio eletroquímico. Neste trabalho, essa variável é utilizada como uma medida complementar ao pH.
		\stophead

		\starthead{Hidrogênio}
			\indentitem O sensor de hidrogênio fornece a variável de interesse utilizada no treinamento do algoritmo LSTM, permitindo posteriormente a substituição do sensor físico por um sensor virtual.
		\stophead

		\starthead{Metano}
			\indentitem Assim como o sensor de hidrogênio, o sensor de metano fornece dados para o treinamento do \getbuffer[sens], possibilitando a estimação da concentração desse gás sem a necessidade de medição direta.
		\stophead

		\starthead{Dióxido de Carbono}
			\indentitem O sensor de dióxido de carbono permite monitorar um dos principais produtos do processo fermentativo. Em conjunto com os demais sensores de gases, seus dados fornecem informações sobre o metabolismo dos microrganismos e sobre as rotas metabólicas predominantes durante a fermentação.

			Quanto maior a representatividade das variáveis monitoradas e mais abrangente for a instrumentação do processo, maior tende a ser a capacidade dos \plural{\getbuffer[sens]} de inferir corretamente as variáveis não medidas.
		\stophead
	\stopitemize

	Todos os dados temporais coletados {\emph in-line} são enviados ao servidor e armazenados de forma sincronizada para cada instante de aquisição. Posteriormente, esses dados são utilizados nas etapas de pré-processamento e treinamento do modelo.
\stopsection
